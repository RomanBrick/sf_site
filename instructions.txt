Инструкции для команды:
1. Репозиторий распологается на GitHub 
2. Изменения вносятся в рабочие ветки не в main 
3. Все изменения и слияния с main проходят через пулл рэквэст с назначением Reviewers вашего Лида
4. Глобальные измения должны быть применены к каждому локальному репозитарию через
git config --global user.name "Напишиет свое имя"
git config --global user.email "Укажите свою почту"
git config --global alias.history "log --graph" - добавляем элиас для просмотра истории измений
Файл с конфигурацией лежит https://github.com/RomanBrick/sf_site/blob/gitsf/.gitconfig
5.
Создать свою ветку:
git checkout -b "Имя вашей будущей ветки"
Добавяте файл на которым работает 
Примените команду для индексирования файла git add . или указите после add имя файла
Сделайте коммит - git commit -m"Укажите измения который произошли"
Залейти измения в вашу ветку git push origin "имя вашей ветки"
После одобрения измений сделайте git rebase main
